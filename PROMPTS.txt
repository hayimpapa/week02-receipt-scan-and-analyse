## Week 2 - Receipt Analyser - Initial Prompt

Build a "Receipt Analyser" web app using React.
This is Week 2 of "52 apps in 52 weeks before
I turn 52" by Hey I'm Papa.

This is a standalone React app with its own
GitHub repository, deployable to Vercel.

CORE FLOW:
1. User opens app on phone or desktop
2. Camera view appears (use device camera on
   mobile, webcam on desktop for testing)
3. User captures a photo of a receipt
4. App sends image to Claude AI vision API
   (claude-sonnet-4-20250514) for extraction
5. Extracted data displays for review and editing
6. User confirms and saves to Supabase
7. Basic reporting view shows spending over time

CAMERA CAPTURE:
- Use browser MediaDevices API for camera access
- On mobile default to rear camera
- On desktop use webcam (for testing)
- Show live camera preview
- Capture button takes the photo
- Show captured image preview before processing
- Allow retake if not happy with the image

CLAUDE AI EXTRACTION:
Use the Anthropic API with vision to extract
the following fields from the receipt image:

- Merchant name
- Date of purchase (format: DD/MM/YYYY)
- Line items, each containing:
  - Product name
  - Quantity
  - Unit price
  - Total price
  - GST amount (if shown)
  - Suggested category (from the list below)
- Receipt total
- Total GST
- Payment method (if visible)

CATEGORIES (fixed list, AI pre-selects,
user can override via dropdown):
Groceries, Fruit & Veg, Meat & Seafood,
Deli & Bakery, Dairy & Eggs, Frozen Foods,
Snacks & Confectionery, Beverages, Alcohol,
Household & Cleaning, Health & Beauty,
Baby & Kids, Pet Supplies, Clothing,
Electronics, Dining Out, Fuel, Other

Prompt Claude to return ONLY valid JSON
in this structure:
{
  "merchant": "",
  "date": "",
  "paymentMethod": "",
  "receiptTotal": 0,
  "totalGST": 0,
  "items": [
    {
      "productName": "",

      "gst": 0,
      "suggestedCategory": ""
    }
  ]
}

## Week 2 - Receipt Analyser - Update 1 - Owner Auth and Proxy

Add a secure owner mode to the Receipt
Scanner & Analyser app with two modes:
Guest and Owner.

GUEST MODE (default, no login):
- Full UI is visible and explorable
- When user clicks Scan or Capture,
  show a friendly message instead:
  "This live demo requires owner login.
  Want to run it yourself? See README.md
  for local setup instructions."
- Supabase warning message stays as is
- No other restrictions

OWNER MODE (password protected):
- Small, subtle lock icon button in the
  top right corner of the app
- Clicking it opens a minimal password
  modal/overlay
- Owner enters password, clicks Unlock
- If correct, session token is stored
  in sessionStorage (clears when browser
  tab closes)
- In owner mode, full scanning and
  Supabase saving works normally
- Lock icon changes to indicate
  logged-in state (e.g. unlocked icon)
- No logout needed - closing the tab
  ends the session

SERVERLESS PROXY FOR ANTHROPIC API:
Create a Vercel serverless function at:
api/scan.js

This function should:
- Accept a POST request with the
  receipt image (base64) in the body
- Check for a valid session token
  in the request header
- If token invalid or missing,
  return 401 with message:
  "Unauthorised - owner login required"
- If token valid, call the Anthropic
  API with the image using the
  ANTHROPIC_API_KEY environment variable
  (server-side only, never exposed
  to the browser)
- Return the extracted receipt JSON
  to the frontend

PASSWORD VALIDATION:
Create a second serverless function at:
api/auth.js

This function should:
- Accept a POST request with the
  password in the body
- Compare against the OWNER_PASSWORD
  environment variable (server-side only)
- If correct, return a signed session
  token (a simple UUID or hash is fine)
- If incorrect, return 401

The frontend stores this token in
sessionStorage and sends it as a
header with every request to api/scan.js

ENVIRONMENT VARIABLES NEEDED
(server-side only, never in .env
or exposed to browser):
- ANTHROPIC_API_KEY
- OWNER_PASSWORD
- SESSION_SECRET (a random string
  used to sign tokens)

These must only ever be used in the
serverless functions, never imported
into any frontend React code.

Update README.md to include:
- Explanation of guest vs owner mode
- List of required Vercel environment
  variables and where to add them
- Note that no .env file is needed
  for the live deployment, only for
  local development

## Week 2 - Receipt Analyser - Update 2 - Move Supabase server-side

Move all Supabase database calls from
the frontend React code into the
existing serverless function api/scan.js

GOAL:
No Supabase keys should exist in any
frontend code or VITE_ environment
variables. All database operations
happen server-side only.

ENVIRONMENT VARIABLES TO USE
(server-side only, already in Vercel):
- SUPABASE_URL
- SUPABASE_ANON_KEY

CHANGES NEEDED:

1. UPDATE api/scan.js
After successfully extracting receipt
data from Anthropic API, also save
the result to Supabase in the same
serverless function.
Save to two tables as before:
Table: receipts
Fields: merchant, date, receipt_total,
total_gst, payment_method, created_at
Table: receipt_items
Fields: receipt_id (foreign key),
product_name, quantity, unit_price,
total_price, gst, category, created_at
Return both the extracted receipt JSON
AND the saved receipt id back to the
frontend so it can display the results.
Install @supabase/supabase-js in the
serverless function if not already
available.

2. CREATE api/receipts.js
A new serverless function to handle
fetching saved receipts for the
Reports tab.
This function should:
- Accept a GET request with optional
  date range query parameters:
  ?range=7d or 30d or 90d or all
- Check for valid session token
  in request header (same auth
  as api/scan.js)
- Query Supabase for receipts
  and their items within the
  date range
- Return the data as JSON
  to the frontend

Remove all direct Supabase client
calls from React components.
Remove VITE_SUPABASE_URL and
VITE_SUPABASE_ANON_KEY from

VITE_SUPABASE_URL
VITE_SUPABASE_ANON_KEY

anywhere in frontend code.
